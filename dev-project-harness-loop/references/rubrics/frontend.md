# Frontend rubric (example)

Goal: make subjective quality gradable.

## Categories

### Design quality (weighted high)
Does the design feel like a coherent whole rather than a collection of parts?
- typography hierarchy
- spacing consistency
- layout rhythm
- color system coherence

### Originality (weighted high)
Evidence of deliberate custom decisions vs template/library defaults.
- avoids generic “AI slop” patterns
- a human designer can point to intentional choices

### Craft
Technical execution fundamentals.
- contrast ratios
- responsive behavior
- interaction states
- accessibility basics

### Functionality
Usability independent of aesthetics.
- primary actions discoverable
- workflows make sense
- no dead-end screens

## Threshold rule
If Design quality or Originality is below threshold → FAIL (or require a pivot).
